- **Fix still needed:** Add state cleanup after timeout in `solo-dev.sh`

### Pattern 2: Pre-Start Instability

- **Occurrences:** 6 aborted starts (01:26-01:51) before successful pipeline
- **Root cause:** Rate limiting (CLI exit code 130) + configuration issues
- **Wasted:** ~25 min of startup overhead
- **Fix:** Add pre-flight validation in `solo-dev.sh`

### Pattern 3: Deploy Scope Confusion (Run 1 only — SELF-CORRECTED)

- **Occurrences:** 1 iteration (Run 1, iter 2)
- **Root cause:** Deploy skill designed for web/cloud. Used AskUserQuestion on CLI/TUI project.
- **Wasted:** 1 iteration (~3 min)
- **Self-corrected:** Deploy clean in all subsequent runs (Runs 10, 12, 13, 14 all no-stall)

### Pattern 4: Valid Redo Cycle (working as designed)

- **Occurrences:** 1 redo cycle (Run 1, iters 4-7)
- **Root cause:** Review correctly identified UTF-8 slicing bug, missing README, no pre-commit hooks
- **Assessment:** Quality catch, not waste

### Pattern 5: Doubled Signal (cosmetic)

- **Occurrences:** Multiple deploy logs show `<solo:done/>` output twice
- **Impact:** None — purely cosmetic

### Pattern 6: Minor Acceptance Criteria Gap (Run 13)

- **Occurrences:** 1 (live-bookmarks spec: `supervox calls` bookmark count in text table)
- **Root cause:** Build completed all 17 plan tasks but plan didn't include a task for CLI table column
- **Impact:** Minimal — data accessible via `--json` output
- **Assessment:** Spec-plan alignment gap, not a build failure

### Pattern 7: CI Cross-Compile Failure (Run 14 — NEW)

- **Occurrences:** 1 (whisper-stt deploy stage)
- **Root cause:** `ort-sys` has no prebuilt binary for `x86_64-apple-darwin` target. Release workflow included both aarch64 and x86_64, causing CI failure.
- **Impact:** Deploy took 26 min (vs ~8 min avg) to diagnose and fix CI config
- **Resolution:** Dropped x86_64 target from release workflow. Acceptable — Apple Silicon only for now.
- **Assessment:** Correct resolution, but /deploy should detect target compatibility earlier

## Recommendations

1. **[CRITICAL]** Fix state leak on timeout in `solo-dev.sh` — add state cleanup after TIMEOUT log entry. This caused action-tracker to be falsely archived (Pattern 1). The defect was worked around but not fixed at source.

2. **[HIGH]** Deploy skill needs CLI/TUI project-type detection — auto-detect from Cargo.toml binary targets + CLAUDE.md keywords. Prevents AskUserQuestion stalls (Pattern 3). Self-corrected in later runs but should be formalized.

3. **[HIGH]** Deploy skill should detect target architecture compatibility — check for native C deps (whisper-rs, ort-sys) that lack cross-compile support before configuring CI targets (Pattern 7).

4. **[MEDIUM]** Add pre-flight validation to `solo-dev.sh` — check API key, rate limit status before entering main loop (Pattern 2).

5. **[MEDIUM]** Improve spec-to-plan alignment — `/plan` should generate a task for every spec criterion. Pattern 6 (live-bookmarks) shows a gap where spec had a criterion not covered by any plan task.

6. **[MEDIUM]** Debug-mode test compilation with whisper feature — `cargo test --workspace` SIGKILL from whisper-rs linking in debug mode. Consider split-test strategy: `cargo test -p supervox-agent && cargo test -p voxkit` in CI instead of workspace-level.

7. **[LOW]** Consider `cargo-tarpaulin` or `cargo-llvm-cov` for coverage metrics.

8. **[LOW]** Add `cargo-audit` in CI for dependency vulnerability scanning.
