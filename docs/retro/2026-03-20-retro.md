## Recommendations

1. **[CRITICAL]** Fix state leak on timeout in `solo-dev.sh` — add state cleanup after TIMEOUT log entry. This caused action-tracker to be falsely archived (Pattern 1). The defect was worked around but not fixed at source.

2. **[HIGH]** Deploy skill needs CLI/TUI project-type detection — auto-detect from Cargo.toml binary targets + CLAUDE.md keywords. Prevents AskUserQuestion stalls (Pattern 3). Self-corrected in later runs but should be formalized.

3. **[MEDIUM]** Add pre-flight validation to `solo-dev.sh` — check API key, rate limit status before entering main loop (Pattern 2).

4. **[MEDIUM]** Improve spec-to-plan alignment — `/plan` should generate a task for every spec criterion. Pattern 6 (live-bookmarks) shows a gap where spec had a criterion not covered by any plan task.

5. **[LOW]** Consider `cargo-tarpaulin` or `cargo-llvm-cov` for coverage metrics. 224 tests is strong but coverage % unknown.

6. **[LOW]** Add `cargo-audit` in CI for dependency vulnerability scanning.
